## Blog of  OOP pillars:

We are already know OOP four pillars Inheritance, Polymorphism, Abstraction and Encapsulation. I am explain in bellow,

1. Inheritance: Allows classes to use extends to share and reuse logic from a parent class, reducing redundancy.

2. Polymorphism: Lets different classes share the same method name but provide unique implementations, making your code more flexible.

3. Abstraction: Uses interfaces or abstract classes to hide complex implementation details, showing only what’s necessary to the user.

4. Encapsulation: Uses access modifiers (private, protected, public) to hide 2. internal logic and protect the state of an object.
